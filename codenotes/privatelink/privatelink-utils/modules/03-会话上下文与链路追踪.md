# 模块 03：会话上下文与链路追踪

## 概述

该模块对应图分析 **Community 2「Session Context」**（4 个节点）和 **Community 5「Session Context Keys」**（4 个节点），提供 `context.Context` 中 Session ID 的存取工具，以及基于上下文的 HTTP 请求方法。该模块贯穿两个源文件——`ncontext/ncontext.go` 提供底层上下文操作，`httpclient/httpclient.go` 中的 `PostCtx` 系列方法在此基础上构建链路日志追踪。

## 源文件

- **`ncontext/ncontext.go`** — 上下文工具实现（31 行）
- **`httpclient/httpclient.go`** — 上下文感知 HTTP 方法（`PostCtx` / `PostCtxWithHeader` / `PostCtxWithZkPath`）

## Context Key 设计

```go
type contextKey int

var (
    SessionID       = "session_id"       // 日志字段名
    KeyForSessionID = contextKey(100)    // context value key
)
```

**设计要点**：
- `contextKey` 是未导出的整数类型，防止外部包通过相同 key 值覆盖
- `SessionID` 是字符串常量 `"session_id"`，供 `zap` 日志字段使用
- `KeyForSessionID` 的值固定为 `100`

**图分析归属**：Community 5「Session Context Keys」。该社区包括 `contextKey` 类型、文件节点 `ncontext.go`，以及 `SessionIDToContext()` 和 `NewSessionIDToContext()` 函数——它们都具有"将 key 写入 context"的共同模式。

## ncontext 工具函数

### SessionIDToContext — 存储 Session ID (`ncontext.go:17-19`)

```go
func SessionIDToContext(ctx context.Context, sessionID string) context.Context {
    return context.WithValue(ctx, KeyForSessionID, sessionID)
}
```

将外部传入的 session ID 存入 context。适用于从 HTTP 请求头或 gRPC metadata 中提取已有 session ID 后继续传递。

### NewSessionIDToContext — 生成并存储 (`ncontext.go:21-23`)

```go
func NewSessionIDToContext(ctx context.Context) context.Context {
    return context.WithValue(ctx, KeyForSessionID, ucontext.NewRequestID())
}
```

调用 `ucontext.NewRequestID()` 生成一个新的唯一请求 ID 作为 session ID。适用于请求入口处初始化链路追踪上下文。

### ExtractSessionIDFromCtx — 提取 Session ID (`ncontext.go:25-31`)

```go
func ExtractSessionIDFromCtx(ctx context.Context) string {
    val := ctx.Value(KeyForSessionID)
    if val == nil {
        return ""
    }
    return val.(string)
}
```

从 context 中提取 session ID。若不存在则返回空字符串（而非 panic）。该函数被 `httpclient` 模块的 `PostCtxWithHeader` 和 `PostCtxWithZkPath` 调用。

**图分析归属**：Community 2「Session Context」。该社区还包括 `httpclient` 中所有使用 context 参数的 `PostCtx` 系列方法——它们在调用链中形成"上下文感知请求"的逻辑集群。

## 上下文感知 HTTP 方法

以下方法位于 `httpclient/httpclient.go`，属于 Community 2 和 Community 3 的交叉区域。它们不直接操作 `ncontext` 的函数，但通过 `ExtractSessionIDFromCtx` 提取 session ID 用于日志。

### PostCtx — 基础上下文 POST (`httpclient.go:137-139`)

```go
func (h *HTTPClient) PostCtx(ctx context.Context, url string, req, resp any, timeout ...uint32) error {
    return h.PostCtxWithHeader(ctx, url, nil, req, resp, timeout...)
}
```

最简的上下文 POST 方法。无自定义 Header，直接委托给 `PostCtxWithHeader`。**ctx 必须已被 `ucontext.ToLoggerContext` 封装过**，否则日志提取会失败。

### PostCtxWithHeader — 带自定义 Header 的上下文 POST (`httpclient.go:162-196`)

```go
func (h *HTTPClient) PostCtxWithHeader(ctx context.Context, url string, header map[string]string, req, resp any, timeout ...uint32) error
```

核心的上下文感知方法。执行流程：

1. **提取 session ID**：`ncontext.ExtractSessionIDFromCtx(ctx)` 从 context 中获取当前会话 ID
2. **构建日志器**：`ucontext.Logger(ctx).With(zap.String(ncontext.SessionID, sessionID))`——将 session ID 注入 zap 日志字段
3. **序列化请求体**：`json.Marshal(req)`
4. **合并 Header**：将 Content-Type 固定为 `application/json`，合并用户自定义 header
5. **日志请求**：`l.Infow("http request", "header", hdr, "url", url, "body", string(b))`
6. **发送请求**：调用 `h.PostBytes(hdr, url, b, timeout...)`
7. **校验状态码**：非 200 状态码返回错误
8. **日志响应**：`l.Infow("http response", "resp", string(httpResp.Body))`
9. **反序列化**：`json.Unmarshal(httpResp.Body, resp)`

### PostCtxWithZkPath — 通过 ZK 服务发现 (`httpclient.go:143-158`)

```go
func (h *HTTPClient) PostCtxWithZkPath(ctx context.Context, zkCli *zkutils.ZKClient, zkPath, path string, hdr map[string]string, req, resp any, timeout ...uint32) error
```

通过 ZooKeeper 进行服务发现后发送 POST 请求。执行流程：

1. **构建日志器**：同上，提取 session ID 构建结构化日志
2. **创建 ZK 解析器**：`zkresolver.NewNameResolver(zkresolver.NewZKPlugin(zkCli, &uflog.Logger{SugaredLogger: l}))`
3. **构造 ZK 路径**：`"zk:///" + zkPath`——注意是三个斜杠
4. **随机解析地址**：`zk.ResolveRandom(wraper)` 从 ZK 中随机获取一个可用的后端地址
5. **地址校验**：若解析为空，记录错误并返回 `fmt.Errorf("resolve zk path empty, %s", zkPath)`
6. **构造目标 URL**：`fmt.Sprintf("http://%s/%s", addr, path)`——路径部分直接拼接
7. **委托发送**：`h.PostCtxWithHeader(ctx, url, hdr, req, resp, timeout...)`

## Session ID 传递流程

```
                  外部请求入口
                       │
                       ▼
           NewSessionIDToContext(ctx)    ← 生成新 session ID
              或 SessionIDToContext(ctx, id)
                       │
                       ▼
         ucontext.ToLoggerContext(ctx)   ← 封装日志上下文
                       │
                       ▼
           PostCtx(ctx, url, req, resp)
                       │
                       ▼
          PostCtxWithHeader(ctx, url, hdr, req, resp)
                       │
               ├── ncontext.ExtractSessionIDFromCtx(ctx) → "xxx"
               ├── ucontext.Logger(ctx) → zap.Logger
               └── l.With(zap.String("session_id", "xxx"))
                       │
                       ▼
              日志记录 (Infow / Errorw)
              字段: "session_id" = "xxx"
```

## 使用示例

```go
import (
    "git.ucloudadmin.com/unetworks/privatelink/privatelink-utils/ncontext"
    "git.ucloudadmin.com/unetworks/privatelink/privatelink-utils/httpclient"
    "git.ucloudadmin.com/cnat2/app/ucontext"
)

// 1. 在请求入口处初始化上下文
ctx := context.Background()
ctx = ncontext.NewSessionIDToContext(ctx)
ctx = ucontext.ToLoggerContext(ctx)  // 封装日志上下文

// 2. 使用上下文感知 POST
type Req struct { Data string `json:"data"` }
type Resp struct { ID string `json:"id"` }

cli := httpclient.NewHTTPClient(&httpclient.Config{Timeout: 10})
err := cli.PostCtx(ctx, "http://api.example.com/v1/resource", &Req{Data: "hello"}, &Resp{})

// 3. 或者通过 ZK 服务发现
// zkCli := zkutils.NewZKClient(...)
// err = cli.PostCtxWithZkPath(ctx, zkCli, "/path/to/service", "api/v1/resource", nil, &Req{}, &Resp{})
```
