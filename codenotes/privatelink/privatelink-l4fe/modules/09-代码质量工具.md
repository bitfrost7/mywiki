## CI 测试

### CodeTest (`.gitlab-ci.yml:102-111`)

```yaml
CodeTest:
  stage: Test
  image: hub.ucloudadmin.com/l4fwd-s/golangbuild:1.20-v2
  before_script:
    - go mod download
  script:
    - go test -short $(go list ./... | grep -v /vendor/)
    - go test -race $(go list ./... | grep -v /vendor/)
    - go test -coverpkg ./... -v -coverprofile .Coverage.cov && go tool cover --func=.Coverage.cov
```

执行三个测试阶段：
1. `go test -short` — 短测试
2. `go test -race` — 竞态检测
3. `go test -coverpkg` — 覆盖率统计

触发条件：仅 MR 事件或 master 分支（`mr_only_template`）。

### CodeLint (`.gitlab-ci.yml:114-121`)

```yaml
CodeLint:
  stage: Test
  image: hub.ucloudadmin.com/l4fwd-s/golangbuild:1.20-v2
  script:
    - golangci-lint run --timeout=5m
```

使用 golangci-lint 进行代码检查，超时 5 分钟。

## Linter 配置

`.golangci.yml:1-67` 启用了 30+ 个 linter，包括：

### 启用的关键 linter
- `errcheck` — 错误检查
- `gosimple` — 代码简化
- `govet` — 基础检查（含 shadowing、unreachable 等子检查）
- `ineffassign` — 无效赋值
- `staticcheck` — 静态分析
- `unused` — 未使用代码
- `goconst` — 重复常量
- `gocritic` — 代码风格
- `gocyclo` — 圈复杂度
- `gofmt`/`gofumpt`/`goimports` — 格式化
- `misspell` — 拼写检查
- `prealloc` — 切片预分配
- `revive` — Go 代码检查
- `whitespace` — 空白检查
