# 模块 05：数据库查询操作层 — Database Query Operations

## 概述

该模块对应图分析 **Community 4「Database Query Operations」**（30 个节点，内聚度 0.09），是数据库查询操作的**聚合中心和桥接模块**。它通过 `Query` 结构体将四个 DAO 聚合在一起，并提供事务、读写分离、上下文管理等基础设施。

## 核心结构体

```go
type Query struct {
```

## 关键函数

### Use() (`gen.go:18-26`)

**核心工厂函数**，是所有 DAO 的入口。

```go
func Use(db *gorm.DB, opts ...gen.DOOption) *Query
```

- 接收 `*gorm.DB` 连接
- 创建四个 DAO 实例并组装到 `Query` 结构
- 在 `mygorm/database.go:New()`（行 51）中被调用：`return db.Use(gormdb), nil`

### Transaction() (`gen.go:87-89`)

```go
func (q *Query) Transaction(fc func(tx *Query) error, opts ...*sql.TxOptions) error
```

包装 `gorm.DB.Transaction()`，自动克隆 DAO 到事务上下文中。

**在 `api/grpc_api.go:205` 中被使用**，确保三张表的查询在同一个事务中执行。

### WithContext() (`gen.go:78-85`)

```go
func (q *Query) WithContext(ctx context.Context) *queryCtx
```

创建绑定上下文的查询对象，每个方法调用都会传递 context。

### Begin / Commit / Rollback (`gen.go:91-115`)

提供手动事务控制：

```go
tx := q.Begin()
// ... 操作 ...
tx.Commit()    // gen.go:101-103
tx.Rollback()  // gen.go:105-107
```

同时支持 `SavePoint` 和 `RollbackTo` 保存点功能。

### ReadDB / WriteDB (`gen.go:53-59`)

```go
func (q *Query) ReadDB() *Query   // 添加 dbresolver.Read clause
func (q *Query) WriteDB() *Query  // 添加 dbresolver.Write clause
```

通过 `gorm.io/plugin/dbresolver` 插件支持读写分离。
