## 多阶段 Docker 构建

`Dockerfile`（典型 x86_64 构建）：

```dockerfile
# 构建阶段
FROM hub.ucloudadmin.com/l4fwd-s/golangbuild:1.20-v2 AS build-env
ADD . /src
WORKDIR /src
RUN CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build \
    -o privatelinkl4fe -ldflags "${LD_VERSIONFLAG}" ./cmd/privatelinkl4fe/*.go

# 运行阶段
FROM hub.ucloudadmin.com/cnat2/alpine:3.9
WORKDIR /app
COPY --from=build-env /src/privatelinkl4fe /app/
```

构建特性：
- **Go 1.20 构建镜像**（`golangbuild:1.20-v2`）
- **静态编译**（`CGO_ENABLED=0`）
- **版本信息注入**：通过 ldflags 注入 Prometheus version 信息（Version、Revision、Branch、BuildDate、BuildUser）
- **Alpine 3.9 运行镜像**：极小基础镜像
- 单独的 `Dockerfile-amd64` 和 `Dockerfile-arm64` 支持多架构

## CI/CD 流水线

### 流水线阶段

```
Test ──► Package
```

### 构建模板 (`.buildimage`)

`.gitlab-ci.yml:19-36` 定义了通用构建模板：

```yaml
.buildimage:
  stage: Package
  image: hub.ucloudadmin.com/public/uaek-kaniko-executor:v1.5.0
  script:
    - /kaniko/executor -c $CI_PROJECT_DIR -f Dockerfile-${ARCH_NAME} -d $IMAGE_NAME:$IMAGE_TAG
```

使用 **Kaniko**（而非 Docker-in-Docker）构建镜像，无需特权模式。

### 构建任务

#### 自动构建（仅 tags）

| Job | 架构 | 模板 |
|-----|------|------|
| `amd64-buildimage` | amd64 | `.buildimage` |
| `arm64-buildimage` | arm64 | `.buildimage`（使用 ARM 版 Kaniko） |

镜像标签规则：`$CI_COMMIT_SHORT_SHA-{amd64,arm64}`，tag 推送时使用 `$CI_COMMIT_TAG`。

#### 手动构建

| Job | 架构 | 模板 |
|-----|------|------|
| `amd64-buildimage-manual` | amd64 | `.buildimage-manual`（`when: manual`） |
| `arm64-buildimage-manual` | arm64 | `.buildimage-manual`（`when: manual`） |

#### 测试构建

`TestBuildImage`：仅在 master 分支触发，用于验证构建流程。

### 触发规则

```yaml
.master_only_template: &master_only_template
  rules:
    - if: '$CI_COMMIT_BRANCH == "master"'

.mr_only_template: &mr_only_template
  rules:
    - if: '$CI_PIPELINE_SOURCE == "merge_request_event"'
    - if: '$CI_COMMIT_BRANCH == "master"'
```
