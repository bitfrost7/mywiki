```
│   └── db.CreateWhitelist()  → INSERT t_service_whitelist                   │

│ │   └── fac.URecycle.NotifyRecoveredEndpoint()  → URecycleV2               │
│ │                                                                            │
│ └────────────────────────────────────────────────────────────────────────────┘
│                                    │
│ ┌── 阶段 6c: 删除 ──────────────────────────────────────────────────────────┐
│                                                                            │
│ │  Cron (ToDeleteSpec) 或 API DeleteEndpoint → recycle.DeleteEndpoint()    │
│ │   ├── fac.URecycle.GetToDeleteEndpointList()  → URecycleV2               │
│ │   ├── fac.UResource.IGetResourceById()        → UResource                │
│ │   ├── fac.PrivateLink.DeleteEndpoint()        → POST → apisvr            │
│ │   │                                              (DeleteVPCEndpoint)      │
│ │   └── fac.URecycle.NotifyRecycledEndpoint()   → URecycleV2               │
│ │                                                                            │
│ └────────────────────────────────────────────────────────────────────────────┘
│
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.3 L4 转发配置数据流

```
外部系统 / apisvr → gRPC → privatelink-l4fe.ListL4Gw(object_ids)
    │
    ├── 事务内查询三表:
    │   ├── t_vpc_endpoint:  WHERE service_id IN (...) AND connect_status=1
    │   │                     AND delete_time=0
    │   │   → result: 各 Endpoint 的 IPv4/IPv6、MAC、隧道 ID
    │   │
    │   ├── t_service:      WHERE service_id IN (...) AND delete_time=0
    │   │   → result: Service 的 IP、TunnelID、VNetID
    │   │
    │   └── t_service_snatips: WHERE service_id IN (...) AND delete_time=0
    │       → result: SNAT IPs
    │
    └── 组装 L4GwView:
        ├── foreIPs    = [endpoint_ipv4, endpoint_ipv6]
        ├── fnatIPs    = [snat_ip1, snat_ip2, ...]
        ├── foreGroups   = 每个 endpoint 一个组 (port: 1-65535)
        ├── fnatGroup    = 所有 SNAT IP 组成一个组 (port: 1025-65535)
        ├── backendGroup = Service IP 组成一个组 (port: 1-65535)
        └── rules      = 每个 fore endpoint → fullnat → fnat + backend
```

源码：`privatelink-l4fe/api/grpc_api.go:32-191`

## 4. 数据同步时序图

### 4.1 流量计费数据通道

```
时间线
    │
    ├── T-1h
    │   billinsert 插入 [T-2h ~ T-1h] 的流量到 t_traffic_info
    │
    ├── T (整点)
    │   billing.TrafficBilling()
    │     ├── SELECT * FROM t_traffic_info WHERE insert_time IN [T-1h, T]
    │     ├── 按 payer 分流汇总
    │     ├── INSERT INTO t_billing_info
    │     └── POST → UBill.PostPaid()
    │
    ├── T+1h
    │   billinsert 插入 [T-1h ~ T] 的流量
    │   billing.TrafficBilling()
    │     └── (重复以上流程)
    │
    └── T + RetryInterval
        billing.TrafficReBilling()
          └── SELECT * FROM t_billing_info WHERE state='fail' FOR UPDATE
              ├── UBill.GetPostPaidItemBillInfo() 查询是否已支付
              ├── if 已支付: UPDATE state='success'
              └── if 未支付: UBill.PostPaid() 重试 + UPDATE state
```

## 5. 关键设计决策（数据流角度）

1. **billinsert → billing 单向数据管道**：`t_traffic_info` 只有 billinsert 写入、billing 读取，形成严格的生产者-消费者模式。避免 billing 直接对接 CloudWatch。

2. **recycle 承担"状态机"角色**：recycle 是唯一写入 `t_vpc_endpoint.close_status` 和 `t_connect_info.end_time` 的服务（除创建时的初始值外）。bwregister 读取这些字段决定限速策略。

3. **全量同步而非增量**：bwregister 每次任务周期全量读取数据库 JOIN 后推送到 UTraffic，适合连接数万级的规模（`bwregister/architecture.md:158`）。

4. **外部系统状态独立**：URecycleV2 持有回收状态（待停服/可恢复/待删除的时间戳），recycle 仅根据外部状态触发本地操作，形成外部状态驱动本地状态更新的模式。

5. **ops 与 apisvr 共享 `t_user_config` 写权限**：两个服务均可写入 `t_user_config`，但 ops 专注于运维 CRUD（Add/Delete/Update），apisvr 在用户 API 流程中也写入此表。

6. **数据库是唯一跨服务数据通道**：除 recycle→apisvr 的 HTTP 删除调用和 bwregister→UTraffic 的 gRPC 同步外，所有服务间的数据交换通过共享 MySQL 数据库完成。无服务间直接事件总线或消息队列。
