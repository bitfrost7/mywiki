## 1. 服务间 API 调用总览

```
┌─────────────┐     HTTP (ZK发现)     ┌──────────────────┐
│   apisvr    │ ──────────────────────→│ privatelink-l4fe │
│ (API Server)│   CreateL4Gw           │ (L4 转发引擎)    │
│             │   DeleteL4Gw           │                  │
│             │   GetGWVIPInfo         │                  │
└──────┬──────┘                       └──────────────────┘
       │
       │ HTTP (Backend="PrivateLink")
       │ DeleteVPCEndpoint / DeleteVPCEndpointService
       ▼
┌──────────────┐
│   recycle    │
│ (带宽回收)    │
└──────────────┘
```

**关键发现**：recycle 通过外部 `Backend="PrivateLink"` 的 HTTP API 执行资源删除——该接口实际由 apisvr 提供。即 **recycle → apisvr** 有隐式调用关系（`recycle/factory/privatelink/basic.go:63-76` → `apisvr` 的 `DeleteVPCEndpoint` / `IDeleteVPCEndpointServiceConfiguration` Action）。

## 3. 同步 vs 异步边界

### 3.1 同步调用（请求-响应）

| 路径 | 协议 | 同步性 | 说明 |
|------|------|--------|------|
| 用户 → **apisvr** | HTTP POST (Action JSON) | 同步 | 用户 API 请求，等待返回 |
| 用户 → **privatelink-ops** | HTTP POST (Action JSON) | 同步 | 运维操作，等待返回 |
| 用户 → **recycle** | HTTP POST (Action JSON) | 同步 | 回收触发请求，等待返回 |
| 外部 → **bwregister** | gRPC | 同步 | 带宽/限速信息查询 |
| 外部 → **privatelink-l4fe** | gRPC | 同步 | L4 网关配置查询 |
| **apisvr** → privatelink-l4fe | HTTP (ZK发现) | 同步 | L4 网关创建/删除/查询 |
| **apisvr** → 外部系统(VPC/LB/UBill/UResource/UAccount) | HTTP POST | 同步 | 资源操作和查询 |
| **billing** → UBill | HTTP POST | 同步 | 扣费和查询 |
| **billinsert** → CloudWatch | HTTP POST | 同步 | 拉取监控指标 |
| **bwregister** → UTraffic | gRPC | 同步 | 带宽信息同步 |
| **recycle** → URecycleV2 | HTTP POST | 同步 | 获取列表/通知状态 |
| **recycle** → UResource | HTTP POST | 同步 | 查询资源信息 |
| **recycle** → apisvr (PrivateLink API) | HTTP POST | 同步 | 删除 Endpoint/Service |
| **privatelink-ops** → UResource | HTTP POST | 同步 | 资源标签操作 |

### 3.2 异步边界（Cron/后台任务）

所有 Cron 驱动的操作构成系统中的异步边界：

```
[时间触发] ──→ [Cron 调度] ──→ [Task 执行] ──→ [同步调用外部系统]
                                                     ↓
                                              [更新 DB 状态]
```

| 路径 | 边界类型 | 延迟 |
|------|----------|------|
| 用户创建 Endpoint → **计费** | Cron 异步（billing 每小时整点） | ≤1h |
| 用户创建 Endpoint → **流量插入** | Cron 异步（billinsert 每小时第2分钟） | ≤1h 2min |
| 用户欠费 → **停服** | Cron 异步（recycle ToClose） | 取决于 ToCloseSpec 配置 |
| 用户欠费停服 → **删除** | Cron 异步（recycle ToDelete, 24h 后） | ≥24h |
| 运维设置用户配置 → **带宽限速生效** | Cron 异步（bwregister SyncUserConfig 60s） | ≤60s |
| **bwregister** 查询 DB → **同步 UTraffic** | Cron 全量同步（10s） | ≤10s |

### 3.3 同步调用中的异步特征

- **billing 重试流程**：同步调用 UBill 失败后标记为失败状态，由后续 cron 重试任务异步重试（`task/rebilling.go`），使用 `SELECT ... FOR UPDATE` 悲观锁防止并发（`billing/architecture.md:147`）
- **billinsert 批量写入**：`CreateTrafficInfo` 使用 `CreateInBatches(infos, 20)` 分批写入，非事务性（`billinsert/architecture.md:118`）

## 4. 详细调用关系

### 4.1 apisvr 调用链

```
用户请求 POST /
    │
    ▼
api.handle() (Action 分发)
    │
    ├──→ db/db.go (CRUD: t_service, t_vpc_endpoint, t_connect_info 等)
    │
    ├──→ factory/vpc/impl.go     → POST → UVPCFEGO (Action: AllocateIp, DescribeVPC 等)
    │
    ├──→ factory/lb/impl.go      → POST → ULB (Action: CreateLoadBalancer 等)
    │
    ├──→ factory/l4/impl.go      → POST → privatelink-l4fe (Action: CreateL4Gw 等)
    │    └── ZK 发现: nr.ResolveRandom("zk:///<zkPath>") 获得 L4 服务地址
    │
    ├──→ factory/uresource/impl.go → POST → UResource (Action: IGetResourceById 等)
    │
    ├──→ factory/uaccount/impl.go  → POST → UAccount (Action: GetUserInfo 等)
    │
    └──→ factory/ubill/impl.go     → POST → UBill (Action: GetPrice)
```

源码：`apisvr/server.go:79-85`

### 4.2 recycling 调用链

```
┌─ 用户 POST / (CloseEndpoint/RecycleEndpoint/DeleteEndpoint)
│
├─ Cron (ToCloseSpec / ToRecoverSpec / ToDeleteSpec)
│
▼
task/recycle_endpoint.go 或 task/recycle_service.go
    │
    ├──→ factory/urecycle/expand.go → POST → URecycleV2
    │    ├── GetToCloseResourceList          (获取待停服列表)
    │    ├── GetToRecoverResourceList        (获取可恢复列表)
    │    ├── GetToDeleteResourceList         (获取待删除列表)
    │    ├── NotifyClosed                    (通知停服完成)
    │    ├── NotifyRecovered                 (通知恢复完成)
    │    └── NotifyRecycled                  (通知回收完成)
    │
    ├──→ factory/uresource/expand.go → POST → UResource
    │    └── IGetResourceById                (查询 RegionId)
    │
    ├──→ factory/privatelink/expand.go → POST → apisvr (Backend="PrivateLink")
    │    ├── DeleteVPCEndpoint               (删除 Endpoint)
    │    └── IDeleteVPCEndpointServiceConfiguration (删除 Service, Force=true)
    │
    └──→ db/db.go (事务更新 t_vpc_endpoint, t_service, t_connect_info)
```

源码：`recycle/task/recycle_endpoint.go:13-122`, `recycle/task/recycle_service.go`

### 4.3 billing 调用链

```
Cron 触发 (每小时整点)
    │
    ▼
task/billing.go
    │
    ├── TrafficBilling (流量计费):
    │   ├── db.GetAllTrafficInfosLastInterval()  → 读 t_traffic_info (billinsert 写入)
    │   ├── db.CreateBillingInfo()               → 写 t_billing_info

    │
    └── InstanceBilling (实例计费):
        ├── db.GetAllConnectInfo()                → 读 t_connect_info
        ├── db.GetAllEndpointPayerInfo()          → 读 t_vpc_endpoint
        ├── db.CreateBillingInfo()                → 写 t_billing_info
        └── fac.UBillImpl.InstancePostPaid()      → POST → UBill (Action: PostPaid)

    ├── db.ReBillingWithOrderPaid()  → 事务内 FOR UPDATE
    ├── UBill.GetPostPaidItemBillInfo()  → 查询是否已支付

源码：`billing/task/billing.go:39-249`, `billing/task/rebilling.go`

### 4.4 billinsert 调用链

```

## 5. 数据流向图

### 5.1 链路追踪上下文传递

```
                    privatelink-utils/ncontext
                           │
              SessionIDToContext / ExtractSessionIDFromCtx
                           │
     ┌─────────────────────┼──────────────────────────┐
     │                     │                          │
     ▼                     ▼                          ▼
  apisvr               billing                   recycle
  (request_uuid +      (session_id 由            (request_uuid +
   session_id)          DB 层使用)                session_id)
     │                     │                          │
     ▼                     ▼                          ▼
  httpclient.PostCtx()  DB 操作日志              httpclient.PostCtx()
  (ncontext.ExtractSessionIDFromCtx)            (ncontext.ExtractSessionIDFromCtx)
```

所有服务通过 `privatelink-utils/ncontext` 实现请求链路追踪。`request_uuid` 和 `session_id` 贯穿 HTTP 调用和 DB 操作日志（`recycle/db/db.go:35-40`, `apisvr/db/db.go:38-43`）。

### 5.2 数据血缘关系

```
[用户创建]                               [监控采集]
    │                                        │
    ▼                                        ▼
apisvr 写入 t_vpc_endpoint          CloudWatch (监控数据)
apisvr 写入 t_service                      │
apisvr 写入 t_connect_info                 │
apisvr 写入 t_service_snatips              ▼
apisvr 写入 t_service_whitelist    billinsert 拉取 → 梯形积分计算
apisvr 写入 t_user_config                     │
                                     写入 t_traffic_info
                                            │
                       ┌────────────────────┤
                       ▼                    ▼
                  billing 读取          bwregister 读取
                  t_billing_info        (t_vpc_endpoint +
                  (写入+更新)             t_service + t_user_config)
                       │                    │
                       ▼                    ▼
                    UBill                UTraffic (gRPC)
                   (扣费)               (带宽限速同步)

[欠费回收]
    │
    ▼
URecycleV2 → recycle → apisvr (删除 Endpoint/Service)
    │
    ▼
更新 t_vpc_endpoint.close_status
更新 t_service.close_status
更新 t_connect_info.end_time
```
