| 外部服务 | 后端名称 | 包路径 | Go 结构体 | 职责 |
|----------|----------|--------|-----------|------|
| **URecycleV2** | `URecycleV2` | `factory/urecycle/` | `URecycleImpl` | 回收系统：获取待处理资源列表 + 通知状态变更 |

- `expand.go` — 业务便利方法（参数组装 → 调用 basic → RetCode 校验 → 返回 error）

## 6. 关键实现逻辑

### 6.1 统一的指标埋点模式

每个子系统的 `APIRequestWithMetrics` 方法实现相同的指标上报模式（以 `urecycle/basic.go:100-112` 为例）：

```
├── ClientRequestSentTotal:  {api="api", backend="URecycleV2", action="NotifyClosed"}  +1
├── ClientResponseReceivedTotal: {api="api", backend="URecycleV2", action="NotifyClosed", code="0"} +1
└── ClientResponseDuration:  Histogram 观察请求延迟 (Buckets: 50,100,200,500,1000,1500,2000,2500,3000ms)
```

### 6.2 URecycleV2 ResourceInfo 与 UResource ResourceInfo 对比

| 字段 | URecycleV2 (`urecycle/basic.go:25-35`) | UResource (`uresource/basic.go:29-44`) |
|------|----------------------------------------|----------------------------------------|
| `ResourceId` | ✅ string（短ID/长ID） | ✅ string（长ID） |
| `RegionId` | ❌ | **✅ int** ← 关键字段，用于过滤 |
| `OrganizationId` | ✅ int | ✅ int |
| `CompanyId` | ✅ (字段名 CompanyId) | `TopOrganizationId`（命名差异） |
| `CloseTime` | ✅ int | ❌ |
| `DeleteTime` | ✅ int | ❌ |
| `VPCId` | ❌ | ✅ string |
| `SubnetId` | ❌ | ✅ string |

**关键设计**：`RegionId` 仅存在于 UResource 的 `ResourceInfo` 中，因此每次回收操作前必须调用 `UResource.IGetResourceInfoById()` 获取 RegionId 进行本 Region 过滤。

### 6.3 统一的 RetCode 校验模式

所有 expand 方法的共同模式：
```go
resp, err := i.BasicMethod(ctx, req)
if err != nil { return nil, err }
if resp.RetCode != 0 { return nil, errors.New(resp.Message) }
return resp, nil
```
