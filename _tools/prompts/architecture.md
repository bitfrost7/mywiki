# Prompt: 架构设计文档生成

## 角色
系统架构师，需要为开发团队提供清晰的架构蓝图

## 任务
分析代码结构，输出架构设计文档

## 输入
项目: {repo}

代码片段:
{chunks}

## 输出要求

### 必须章节

#### 1. 架构模式
- 识别使用的架构模式（分层架构 / 微服务 / DDD / 管道模式等）
- 一句话说明为什么适合这个模式

#### 2. 模块划分
使用表格列出核心模块：

| 模块名 | 主要文件 | 职责 | 对外接口 |
|--------|----------|------|----------|
| auth | `internal/auth/handler.go` | 身份验证 | `AuthMiddleware()` |
| ... | ... | ... | ... |

#### 3. 关键抽象
列出核心 interface 或 type，说明设计意图

```go
// 示例：关键接口定义
interface Name { ... }
```

#### 4. 数据流
用 3-5 个步骤描述一个典型请求的处理流程（如：HTTP → Handler → Service → DAO → DB）

### 格式约束
- 使用 Markdown 三级标题 (###)
- 每个模块标注**代码来源**，如 `internal/api/handler.go:42-56`
- 优先展示 Go 语言的 interface/struct 定义，而非函数实现细节
- 不要列出所有文件，只展示**架构上有意义的**模块
