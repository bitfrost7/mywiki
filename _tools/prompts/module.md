# Prompt: 模块详情文档生成

## 角色
代码审查者，需要深入理解单个模块的实现

## 任务
为特定代码文件/包生成详细文档

## 输入
模块: {module_name}
所属项目: {repo}

代码:
```go
{code}
```

## 输出要求

### 1. 一句话职责
用一句话概括这个模块"做什么"

### 2. 主要符号清单
| 名称 | 类型 | 功能简述 | 行号 |
|------|------|----------|------|
| Handler | struct | HTTP 请求处理器 | 23 |
| Process | func | 核心业务逻辑 | 45 |
| ... | ... | ... | ... |

### 3. 关键实现逻辑
用 2-3 段描述核心算法或业务逻辑：

```markdown
模块采用 [模式名] 模式处理 [问题]。主要流程：
1. 首先验证输入参数，检查 xxx
2. 然后调用 [依赖] 获取 [数据]
3. 最后执行 [操作] 并返回 [结果]

关键设计决策：
- 使用 [结构] 来 [目的]
- 通过 [方式] 处理 [边界情况]
```

### 4. 外部依赖
```markdown
| 依赖模块 | 调用方式 | 用途 |
|----------|----------|------|
| user.DAO | 接口注入 | 查询用户信息 |
| redis.Client | 全局实例 | 缓存 token |
| ... | ... | ... |
```

### 5. 使用示例（如有）
如果代码中有清晰的使用模式，展示：

```go
// 初始化
h := NewHandler(dao, cache)

// 调用
result, err := h.Process(ctx, input)
```

### 约束
- 代码长度超过 100 行时，只分析前 100 行 + 函数签名
- 优先描述"为什么这样设计"而非"代码怎么写"
- 每个符号必须标注**精确行号**
