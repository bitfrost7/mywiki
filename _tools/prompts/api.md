# Prompt: API 文档生成

## 角色
后端开发工程师，需要为前端/客户端团队提供接口文档

## 任务
从 HTTP/gRPC 处理代码中提取接口清单

## 输入
代码片段:
{chunks}

## 输出要求

### 接口清单表格

| 路由 | 方法 | 处理函数 | 文件位置 | 功能简述 |
|------|------|----------|----------|----------|
| /api/v1/users | GET | GetUsers | handler.go:45 | 获取用户列表 |
| ... | ... | ... | ... | ... |

### 关键接口详情
选择 3-5 个最重要的接口，补充：

```markdown
### POST /api/v1/users

**功能**: 创建新用户

**请求参数**:
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 用户名 |
| email | string | 是 | 邮箱 |

**处理逻辑** (从代码理解):
1. 验证参数格式
2. 检查邮箱是否已存在
3. 写入数据库
4. 返回用户 ID

**代码位置**: `internal/api/user_handler.go:67-89`
```

### 路由注册
如果代码中有路由注册（如 `gin.GET(...)`），列出路由分组：

```go
// 注册位置: internal/router/router.go:23
r.Group("/api/v1")
  ├── GET  /users      → userHandler.List
  ├── POST /users      → userHandler.Create
  └── ...
```

### 约束
- 只展示能从代码确认的接口，不猜测未实现的路由
- 每个接口必须标注**代码位置**（文件 + 行号）
- 如果识别出是 gRPC，使用 protobuf 格式描述
