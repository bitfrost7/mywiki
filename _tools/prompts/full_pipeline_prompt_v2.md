# 代码知识库文档生成任务（与 pipeline.py 一致版本）

## 任务目标

为 `privatelink/apisvr` 代码仓库生成人类可读、层次清晰的知识库文档。

## 目标仓库

- **系统**: privatelink
- **仓库**: apisvr
- **路径**: `~/.cache/mywiki-repos/apisvr` 或 `~/Documents/Code/work/privatelink/apisvr`
- **语言**: Go

## 生成策略

按以下顺序执行，每层生成后整合：

---

## 第 1 层：项目概览

**角色**: 资深软件架构师

**输入**: 选取 5 个代表性代码块（浏览仓库后选择）

**Prompt**:
```
你是一个资深软件架构师，需要为代码仓库生成项目概览文档。

【任务】基于以下代码片段，生成简洁的项目概览（中文）：

{粘贴代码块}

【输出要求】
1. 项目名称和一句话描述
2. 核心职责（2-3 点）
3. 技术栈（语言、框架、关键依赖）
4. 目录结构简述

【格式】Markdown，使用二级标题，控制在 300 字以内。
```

**输出文件**: `README.md`

---

## 第 2 层：架构设计

**输入**: 所有 interface/type 定义 + 前 5 个 handler 代码

**Prompt**:
```
基于以下代码信息，生成架构设计文档。

【项目】apisvr
【代码片段】
{粘贴代码块}

【任务】分析并输出：
1. 整体架构模式（如分层架构、微服务、DDD）
2. 核心模块划分及职责
3. 关键接口/抽象定义
4. 数据流概览

【格式要求】
- 使用 Markdown 三级标题组织
- 每个模块用表格列出：模块名 | 文件位置 | 职责
- 关键接口标注代码来源，如 `internal/api/handler.go:42`
```

**输出文件**: `architecture.md`

---

## 第 3 层：API 文档

**输入**: 所有 HTTP handler 代码（最多 15 个）

**Prompt**:
```
分析以下 HTTP/gRPC 接口代码，生成 API 文档。

【代码片段】
{粘贴代码块}

【输出格式】
| 接口路径 | 方法 | 处理函数 | 文件位置 |
|---------|------|---------|----------|
| /api/v1/... | GET | GetUser | handler.go:45 |

【附加说明】对每个关键接口，用 1 句话描述其业务含义。
```

**输出文件**: `api.md`

---

## 第 4 层：数据模型

**输入**: 所有 Model/DAO 相关代码（最多 15 个）

**Prompt**:
```
分析以下数据库模型代码，生成数据模型文档。

【代码片段】
{粘贴代码块}

【输出格式】
| 表名/模型 | 字段 | 类型 | 说明 | 文件位置 |
|----------|------|------|------|----------|

【附加】列出关键 SQL 操作或 DAO 方法。
```

**输出文件**: `db.md`

---

## 第 5 层：模块详情

**输入**: 每个核心文件（最多 10 个文件，每文件取前 3000 字符）

**Prompt**:
```
为以下代码模块生成详细文档。

【模块】{文件名}
【所属】apisvr
【代码】
```go
{粘贴代码}
```

【任务】输出：
1. 模块职责（一句话）
2. 主要类型/函数清单（表格：名称 | 类型 | 功能简述 | 行号）
3. 关键实现逻辑（2-3 段描述）
4. 外部依赖（调用了哪些其他模块）

【格式】Markdown，简洁明了，避免大段代码粘贴。
```

**输出目录**: `modules/`，每文件一个 `.md`

---

## 输出目录结构

```
~/Documents/Code/work/mywiki/CodeNotes/privatelink/apisvr/
├── README.md          # 第 1 层：项目概览
├── architecture.md    # 第 2 层：架构设计
├── api.md             # 第 3 层：API 文档
├── db.md              # 第 4 层：数据模型
└── modules/           # 第 5 层：模块详情
    ├── handler.md
    ├── service.md
    └── ...
```

## 关键约束

1. **代码位置标注**: 每个重要信息都标注 `文件.go:行号`
2. **只描述已观察**: 不猜测未实现的功能
3. **中文表达**: 简洁明了，避免大段代码粘贴
4. **表格优先**: 结构化信息用 Markdown 表格
