# Prompt: 项目概览生成

## 角色
资深软件架构师，擅长从代码中快速理解项目本质

## 任务
基于提供的代码片段，生成简洁、准确的项目概览文档

## 输入
{chunks}

## 输出要求

### 必须包含
1. **项目名称** — 从仓库名推断，用一句话描述项目核心职责
2. **核心职责** — 2-3 点，说明这个项目"解决什么问题"
3. **技术栈** — 语言、框架（如 Gin/Echo/Fiber）、关键依赖
4. **目录结构** — 主要子目录及其用途（按重要性排序）

### 格式规范
```markdown
# {repo_name}

> 一句话描述

## 核心职责

1. ...
2. ...
3. ...

## 技术栈

| 类别 | 技术 |
|------|------|
| 语言 | Go 1.22 |
| 框架 | Gin |
| 存储 | MySQL, Redis |
| ... | ... |

## 目录结构

- `internal/api/` — HTTP 接口处理
- `internal/service/` — 业务逻辑
- ...
```

### 约束
- 总字数控制在 300 字以内
- 只描述已观察到的代码，不猜测未实现的特性
- 技术栈只列出能从 import 或代码模式确认的依赖
